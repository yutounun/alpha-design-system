<Title>Installation</Title>

Source code, `registry.json`, and theme styles live in **[yutounun/alpha-design-system](https://github.com/yutounun/alpha-design-system)** on GitHub. Pick **shadcn CLI** (copy component source into your app) or the **npm package** (`alpha-design-system`).

## Install with shadcn

Use the hosted **shadcn registry** when you want files in your repo (same idea as shadcn/ui) instead of importing from npm.

1. Use a project with the **shadcn** CLI and **Tailwind CSS v4** configured in line with this design system.
2. Add a component by pointing at a registry manifest. Builds publish JSON under:

`https://alpha-design-system-five.vercel.app/r/<name>.json`

Replace `<name>` with a registry item (e.g. `button`, `input`, `utils`). The authoritative list of names is [`registry.json`](https://github.com/yutounun/alpha-design-system/blob/main/registry.json) in the repository.

Example — **Button**:

```bash
npx shadcn@latest add https://alpha-design-system-five.vercel.app/r/button.json
```

Items that use `cn` depend on the **`utils`** registry entry; the CLI should pull it in via `registryDependencies`. If you wire manifests by hand, add **`utils`** before other UI pieces.

**Styles after copy:** Either import tokens from the npm package (see below) or copy [`src/styles/theme.css`](https://github.com/yutounun/alpha-design-system/blob/main/src/styles/theme.css) from the repo into your app and load it from your CSS entry so variables and `@theme` match Storybook.

## Install from npm

Install **alpha-design-system** with your preferred package manager. Peers: **React** 18–19, **React DOM**, and **Tailwind CSS** v4.

```bash
pnpm add alpha-design-system
```

```bash
npm install alpha-design-system
```

```bash
yarn add alpha-design-system
```

## Styles

Load tokens and Tailwind once from your app’s CSS entry. **Vite** and **Next.js** differ: Vite often uses a **copied** `theme.css` from this repo; Next.js typically imports **`alpha-design-system/styles.css`** from the package in the same file as `@import "tailwindcss"`.

### Vite + React

1. Install **Tailwind CSS v4** with the **@tailwindcss/vite** plugin ([Installing Tailwind CSS with Vite](https://tailwindcss.com/docs/installation/using-vite)).
2. **Clean the starter:** delete `src/App.css`, remove `import "./App.css"` from `src/App.tsx`, and clear default rules from `src/index.css` (`:root`, `body`, demo styles).
3. Copy [`src/styles/theme.css`](https://github.com/yutounun/alpha-design-system/blob/main/src/styles/theme.css) into your app as `src/styles/theme.css`. That file already includes `@import "tailwindcss"` and the design tokens — when using this copy, do **not** also `@import "alpha-design-system/styles.css"` in `index.css`.
4. Set `src/index.css` to **only** (paths relative to that file):

```css
@import "./styles/theme.css";
@source "../node_modules/alpha-design-system/dist/**/*.js";
```

If your CSS entry is not `src/index.css`, add or remove `..` in `@source` until it resolves to `node_modules/alpha-design-system/dist/**/*.js`.

Ensure `main.tsx` still imports `./index.css`.

### Next.js / App Router (and package `styles.css`)

In the global CSS entry (e.g. `app/globals.css`):

```css
@import "tailwindcss";
/* Path is relative to THIS file — add/remove `..` until it reaches node_modules */
@source "../node_modules/alpha-design-system/dist/**/*.js";

@import "alpha-design-system/styles.css";
```

Examples (adjust `..` to match layout):

| CSS entry location               | Example `@source`                                                |
| -------------------------------- | ---------------------------------------------------------------- |
| `src/index.css`                  | `@source "../node_modules/alpha-design-system/dist/**/*.js";`    |
| `src/app/globals.css`            | `@source "../../node_modules/alpha-design-system/dist/**/*.js";` |
| `app/globals.css` (project root) | `@source "../node_modules/alpha-design-system/dist/**/*.js";`    |

Keep `@import "tailwindcss"` first. If utilities from the package are missing, verify the relative `@source` path.

### shadcn only (no npm package)

Use the **theme stylesheet** from the repo ([`theme.css`](https://github.com/yutounun/alpha-design-system/blob/main/src/styles/theme.css)) instead of `alpha-design-system/styles.css`. Point `@source` at the **files Tailwind should scan** (for example your copied `components/` tree). Use `@source` for `node_modules/alpha-design-system/dist/**/*.js` only when you **do** install the `alpha-design-system` package and import its pre-built components.

## Components (npm)

Prefer **subpath** imports so the bundler resolves only what you use:

```tsx
import { Button } from "alpha-design-system/button"
```

```tsx
// Use the root barrel only when you import many components:
import { Button } from "alpha-design-system"
```

Example:

```tsx
import { Button } from "alpha-design-system/button"

export function App() {
    return (
        <div className="p-6">
            <Button variant="primary">Click me</Button>
        </div>
    )
}
```

Variants, sizes, loading, and `asChild`: see **Components → Button** in the sidebar. Other published subpaths are listed in the package `exports` field (e.g. `form-field`, `input`, `label`, `tabs`, `textarea`).
