**Alpha Design System** is a token-driven React 19 UI layer for building accessible product interfaces with a calm, consistent rhythm—**design tokens**, **primitives**, and **Storybook docs** in one place.

- **Typography & tokens** — Semantic colors, type, and radius from CSS variables and Tailwind `@theme`, so light/dark and multiple themes stay coherent.
- **Layout-friendly adoption** — Import styles once, compose primitives, and iterate in Storybook without bolting on a separate CSS architecture.
- **React & Storybook first** — Live token galleries, MDX reference, and toolbar theme switching mirror how you’ll run the system in a Vite app.

## Why Alpha Design System?

- **No bespoke CSS architecture required** — Layering, tokens, and naming conventions are centralized in `src/styles/theme.css` and related entrypoints; extend via theme overrides instead of one-off globals.
- **Works by importing CSS** — Ship `theme.css` / `index.css` from your app root; no extra build plugin beyond your bundler’s CSS pipeline and Tailwind 4.
- **Comfortable with React & Storybook** — Components, MDX docs, and **Theme** / **Mode** toolbar globals match how you’ll set `data-theme` and `.dark` in production.

## Zero-build mental model

You can treat the design system as **“import CSS + use components”**. In a Vite + React app, wire the same stylesheet Storybook uses:

```tsx
// e.g. src/main.tsx
import "./styles/index.css"
```

For a plain HTML shell, link the built CSS or duplicate the import graph your app uses—there is no runtime framework inside the token layer.
