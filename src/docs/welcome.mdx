**Alpha Design System** is a collection of UI primitives and design tokens built for React 19, Tailwind CSS 4, and Storybook. It prioritizes accessibility and token-driven styling, with components you can drop straight into a product.

## What you get

- **Consistent tokens** — Color, typography, radius, and more come from CSS variables in `src/styles/theme.css` and Tailwind's `@theme`. Avoid hard-coded colors and `dark:` variants; use semantic tokens for elevation and hover states.
- **Multiple themes** — Use the Storybook toolbar **Theme** (Theme 1 / Theme 2) and **Mode** (Light / Dark) to swap tokens via `data-theme` and `.dark`. Apply the same attributes on the root in a Vite app for the same behavior.
- **Components** — These primitives are available (open each Docs / Stories from **Components** in the sidebar):

| Component     | Summary                                                                                            |
| ------------- | -------------------------------------------------------------------------------------------------- |
| **Button**    | Primary actions. Supports variant / size, `loading`, and `asChild` (composition with links, etc.). |
| **Input**     | Single-line text. Sizes and border / focus ring tied to `aria-invalid`.                            |
| **Textarea**  | Multi-line input. Borders and focus use the same token family as Input.                            |
| **Label**     | Form labels; use with `FormField`.                                                                 |
| **FormField** | Wraps label, hint, and error around a control.                                                     |

## Documentation

- **Design Tokens** — Resolved CSS variables for the current theme, with live previews for typography and radius. Changing the toolbar theme **updates values in place**.
- **Component Docs** — MDX and Canvas for APIs, examples, and interactive controls.

## Adopting in your project (overview)

1. Load `theme.css` (or via `index.css`) from your style entry.
2. Import components from `@/components/...`.
3. When adding classes, follow the repo guardrail (`.cursor/skills/guardrail`) and use tokens and Tailwind primitives only.
