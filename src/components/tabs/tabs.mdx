**Tabs** organise content into switchable panels without a page navigation. Built on [Radix UI Tabs](https://www.radix-ui.com/primitives/docs/components/tabs) — keyboard navigation, ARIA roles, and focus management are all handled by the primitive.

## Usage

```tsx
import { Tabs, TabsList, TabsTrigger, TabsContent } from "@/components/tabs"

export function Example() {
    return (
        <Tabs defaultValue="account">
            <TabsList>
                <TabsTrigger value="account">Account</TabsTrigger>
                <TabsTrigger value="password">Password</TabsTrigger>
            </TabsList>
            <TabsContent value="account">Account panel</TabsContent>
            <TabsContent value="password">Password panel</TabsContent>
        </Tabs>
    )
}
```

## Variants

`TabsList` accepts a `variant` prop that controls the visual style of the tab bar.

| Variant | Description |
|---------|-------------|
| `pills` (default) | Segmented-control look. Active tab lifts as a white card on a muted background. |
| `line` | Underline look. Active tab shows a 2 px primary-coloured bottom border. |

## Motion

The active highlight (pills fill or line underline) is a shared indicator that interpolates `transform` and `width` over **200 ms** when the selected tab changes. In environments with `prefers-reduced-motion: reduce`, transitions are disabled automatically.

## Accessibility

- The tab list is a `role="tablist"` element; each trigger is `role="tab"` and each content panel is `role="tabpanel"`.
- Keyboard: **Arrow keys** move between triggers, **Space / Enter** activates the focused trigger.
- Disabled triggers are properly inert and skipped during keyboard navigation.
- Focus ring is rendered via `focus-visible:ring` — visible only on keyboard navigation, not on pointer click.
