**Chart** follows the same pattern as [shadcn/ui Chart](https://ui.shadcn.com/docs/components/radix/chart): compose **Recharts v3** primitives inside **`ChartContainer`**, and add **`ChartTooltip`** / **`ChartLegend`** when you need them. Pass design tokens such as `var(--color-chart-1)` through **`ChartConfig`** (see `theme.css`).

Consumers must install **`recharts`** (this package lists it as an optional `peerDependency`).

## How `ChartConfig` works

`ChartConfig` maps **data keys** to **labels and colors**.

```ts
type ChartConfig = Record<
    string,
    {
        label?: React.ReactNode
        icon?: React.ComponentType
    } & (
        | { color?: string; theme?: never }
        | { color?: never; theme: Record<"light" | "dark", string> }
    )
>
```

When you pass `config` to **`ChartContainer`**, it injects scoped CSS variables of the form `` `--color-<configKey>` `` (for example `` `--color-desktop` `` for the key `desktop`) for each entry that has `color` or `theme`. On each series, set `stroke` / `fill` to the matching `` `var(--color-<configKey>)` `` so **tooltip, legend, and series colors stay aligned**.

Example wiring:

```tsx
// (1) Row keys in `data` match config keys.
const lineData = [
    { month: "Jan", desktop: 186, mobile: 80 },
    { month: "Feb", desktop: 305, mobile: 200 },
]

// (2) `desktop` / `mobile` define label + source color token.
const lineConfig = {
    desktop: {
        label: "Desktop",
        color: "var(--color-chart-1)",
    },
    mobile: {
        label: "Mobile",
        color: "var(--color-chart-2)",
    },
} satisfies ChartConfig

// (3) Series use `var(--color-{key})` so ChartStyle + tooltip/legend stay in sync.
<Line
    dataKey="desktop"
    stroke="var(--color-desktop)"
    strokeWidth={2}
/>
```

For **pie slices** keyed by a categorical field (e.g. `browser`), set each row’s **`fill`** to `var(--color-chrome)` and add a matching **`chrome`** entry in `chartConfig` (with **`label`** and **`color`**). Point **`ChartTooltipContent`** at that field with **`nameKey`** so labels resolve reliably.

## Color tokens

`theme.css` defines **`--color-chart-1`** through **`--color-chart-5`** for both light and dark. It is common to use those as the **`color`** values inside `ChartConfig`. The generated per-series variables (for example `` `--color-desktop` ``) only apply inside the chart scope, so they do not clash with global page CSS.

### Recommended defaults

- **`ChartContainer`**: always pass **`config`** and a **`className`** that sets a minimum height (`min-h-*`).
- Recharts root (**`LineChart`**, **`BarChart`**, etc.): **`accessibilityLayer`** (see **Accessibility** below).

## Line chart

**When to use:** Multiple series over the same dimension (trend vs comparison).

**Recipe focus:** Two **`Line`** components with different **`dataKey`** values; each **`stroke`** uses **`var(--color-…)`** that matches the config key.

```tsx
import {
    CartesianGrid,
    Line,
    LineChart,
    XAxis,
    YAxis,
} from "recharts"

import {
    ChartContainer,
    ChartLegend,
    ChartLegendContent,
    ChartTooltip,
    ChartTooltipContent,
    type ChartConfig,
} from "@/components/chart"

const lineData = [
    { month: "Jan", desktop: 186, mobile: 80 },
    { month: "Feb", desktop: 305, mobile: 200 },
    { month: "Mar", desktop: 237, mobile: 120 },
    { month: "Apr", desktop: 260, mobile: 190 },
    { month: "May", desktop: 209, mobile: 130 },
    { month: "Jun", desktop: 214, mobile: 140 },
]

const lineConfig = {
    desktop: {
        label: "Desktop",
        color: "var(--color-chart-1)",
    },
    mobile: {
        label: "Mobile",
        color: "var(--color-chart-2)",
    },
} satisfies ChartConfig

export function LineChartExample() {
    return (
        <ChartContainer
            config={lineConfig}
            className="min-h-64 w-full max-w-xl"
        >
            <LineChart accessibilityLayer data={lineData}>
                <CartesianGrid vertical={false} />
                <XAxis
                    dataKey="month"
                    tickLine={false}
                    axisLine={false}
                    tickMargin={8}
                />
                <YAxis
                    orientation="right"
                    tickLine={false}
                    axisLine={false}
                />
                <ChartTooltip content={<ChartTooltipContent />} />
                <ChartLegend content={<ChartLegendContent />} />
                <Line
                    dataKey="desktop"
                    type="monotone"
                    stroke="var(--color-desktop)"
                    strokeWidth={2}
                    dot={false}
                />
                <Line
                    dataKey="mobile"
                    type="monotone"
                    stroke="var(--color-mobile)"
                    strokeWidth={2}
                    dot={false}
                />
            </LineChart>
        </ChartContainer>
    )
}
```

## Pie chart

**When to use:** Part-to-whole breakdowns.

**Recipe focus:** Each row’s **`browser`** is the slice id. **`fill`** uses **`var(--color-chrome)`** etc., matching keys in **`chartConfig`**. Use **`nameKey="browser"`** with **`hideLabel`** on **`ChartTooltipContent`** so slice names resolve correctly.

```tsx
import { Pie, PieChart } from "recharts"

import {
    ChartContainer,
    ChartTooltip,
    ChartTooltipContent,
    type ChartConfig,
} from "@/components/chart"

const pieData = [
    { browser: "chrome", visitors: 275, fill: "var(--color-chrome)" },
    { browser: "safari", visitors: 200, fill: "var(--color-safari)" },
    { browser: "firefox", visitors: 120, fill: "var(--color-firefox)" },
    { browser: "edge", visitors: 90, fill: "var(--color-edge)" },
]

const pieConfig = {
    visitors: { label: "Visitors" },
    chrome: { label: "Chrome", color: "var(--color-chart-1)" },
    safari: { label: "Safari", color: "var(--color-chart-2)" },
    firefox: { label: "Firefox", color: "var(--color-chart-3)" },
    edge: { label: "Edge", color: "var(--color-chart-4)" },

export function PieChartExample() {
    return (
```

## Donut chart

**When to use:** Same as pie, but with room for a center metric or label.

**Recipe focus:** **`innerRadius`** / **`outerRadius`** control ring thickness. Center copy uses **`Label`** inside **`Pie`** with **`content={({ viewBox }) => ...}`**.

```tsx
import { Label, Pie, PieChart } from "recharts"

import {
    ChartContainer,
    ChartTooltip,
    ChartTooltipContent,
    type ChartConfig,
} from "@/components/chart"

const pieData = [
    { browser: "chrome", visitors: 275, fill: "var(--color-chrome)" },
    { browser: "safari", visitors: 200, fill: "var(--color-safari)" },
    { browser: "firefox", visitors: 120, fill: "var(--color-firefox)" },
    { browser: "edge", visitors: 90, fill: "var(--color-edge)" },
]

const pieConfig = {
    visitors: { label: "Visitors" },
    chrome: { label: "Chrome", color: "var(--color-chart-1)" },
    safari: { label: "Safari", color: "var(--color-chart-2)" },
    firefox: { label: "Firefox", color: "var(--color-chart-3)" },
    edge: { label: "Edge", color: "var(--color-chart-4)" },
} satisfies ChartConfig

export function DonutChartExample() {
    return (
        <ChartContainer config={pieConfig} className="min-h-72 w-full max-w-sm">
            <PieChart accessibilityLayer>
                <ChartTooltip
                    content={
                        <ChartTooltipContent nameKey="browser" hideLabel />
                    }
                />
                <Pie
                    data={pieData}
                    dataKey="visitors"
                    nameKey="browser"
                    innerRadius={56}
                    outerRadius={80}
                    strokeWidth={2}
                >
                    <Label
                        content={({ viewBox }) => {
                            if (viewBox && "cx" in viewBox && "cy" in viewBox) {
                                return (
                                    <text
                                        x={viewBox.cx}
                                        y={viewBox.cy}
                                        textAnchor="middle"
                                        dominantBaseline="middle"
                                    >
                                        <tspan
                                            x={viewBox.cx}
                                            y={(viewBox.cy ?? 0) - 6}
                                            className="fill-foreground-high text-lg font-semibold"
                                        >
                                            685
                                        </tspan>
                                        <tspan
                                            x={viewBox.cx}
                                            y={(viewBox.cy ?? 0) + 14}
                                            className="fill-foreground-low text-2xs"
                                        >
                                            visitors
                                        </tspan>
                                    </text>
                                )
                            }
                            return null
                        }}
                    />
                </Pie>
            </PieChart>
        </ChartContainer>
    )
}
```

## Accessibility

Add **`accessibilityLayer`** to the Recharts root (**`LineChart`**, **`BarChart`**, **`AreaChart`**, **`PieChart`**, etc.) to enable keyboard interaction and screen reader summaries. The recipes above and **Recommended defaults** include it.
